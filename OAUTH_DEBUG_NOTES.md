# OAuth Sign-Out Bug - Debugging Notes

## Date: 2026-05-24
## Bug: Second login fails after sign-out

### Symptoms
- First login works correctly
- After sign-out → sign-in again = "Access denied. Your email is not authorized."
- Bug cannot reproduce consistently

### Root Cause Analysis

#### Issue #1: Stale Environment Variable Cache ⚠️
**Problem**: `allowedEmails` was loaded once at module initialization
```typescript
// BEFORE (BUGGY):
const allowedEmails = process.env.INTERNAL_ALLOWED_EMAILS?.split(',')...
```

**Impact**: In serverless environments (Vercel Edge), if the module stays cached, the allowed emails array could become stale or undefined after the first check.

**Fix**: Load dynamically on each sign-in attempt
```typescript
// AFTER (FIXED):
function getAllowedEmails(): string[] {
  const emails = process.env.INTERNAL_ALLOWED_EMAILS?.split(',')...
  return emails
}
```

#### Issue #2: Missing OAuth Flow Reset
**Problem**: Google OAuth provider didn't force fresh consent/account selection on re-login

**Impact**: Could reuse cached OAuth state from previous session

**Fix**: Added `prompt: "select_account"` to force fresh account selection

#### Issue #3: Insufficient Logging
**Problem**: No visibility into:
- Which callback was failing (signIn, jwt, session)
- Cookie state during second login
- Timing of email validation

**Fix**: Added comprehensive logging at every auth checkpoint:
- `[Auth:signIn]` - Email validation logic
- `[Auth:jwt]` - Token creation/refresh
- `[Auth:session]` - Session hydration
- `[Middleware]` - Cookie presence and session state

#### Issue #4: No Explicit Sign-Out Callback
**Problem**: `signOut()` called without parameters might not properly clear all state

**Fix**: Added explicit redirect and callback:
```typescript
signOut({ 
  callbackUrl: '/',
  redirect: true 
})
```

### Additional Notes

- NextAuth 5 (beta) uses JWT strategy by default
- Edge middleware can cache aggressively - dynamic loading prevents stale data
- Google OAuth `prompt` parameter forces fresh consent = clean state
- Vercel Edge Functions have cold starts that can affect module caching

### If Bug Still Occurs

Check Vercel logs for:
1. Is `getAllowedEmails()` returning empty array?
2. Is env var properly set in Vercel?
3. Are cookies being set with correct domain/path?
4. Is there a timing issue between sign-out and sign-in?

Add temporary debug logging in production:
```typescript
console.log('[DEBUG] Raw env:', process.env.INTERNAL_ALLOWED_EMAILS)
console.log('[DEBUG] Parsed emails:', allowedEmails)
```
