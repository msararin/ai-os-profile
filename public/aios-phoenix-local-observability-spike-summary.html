Why We Did Not Install Phoenix Immediately

AIOS did not start with the tool. It first defined what evidence would count: no-platform baseline, fixture catalog, 13-slot smoke-test catalog, 23-parameter benchmark matrix, checker/gate-runner v0.1, claim-boundary checks, public-safety checks, and provider-review gates with receipts retained in the internal evidence packet.

This avoided treating a dashboard or tool install as proof. Phoenix was used only after the evidence shape and claim boundaries were frozen.

What Problems We Faced

  • A dashboard could be mistaken for a source of truth.
  • Local traces could be overread as production telemetry.
  • A 12 PASS + 1 WARN smoke result could be compressed into the unsafe claim that all 13 smoke tests passed.
  • Fixture-level evidence could be overread as full runtime authority enforcement.
  • Simulated Big Crew labels could be mistaken for independent multi-worker dispatch.
  • Tool adoption could consume time before proving evidence value.
