## Change Protocol

**When adding new approved concept:**
1. Add to this registry
2. Mark status: Present / Incomplete / Missing
3. Commit registry update
4. This becomes new baseline

**When superseding approved concept:**
1. Mark old concept as "superseded by [new concept]"
2. Add new concept to registry
3. Document reason for supersession
4. Get Lyn approval
5. Then remove old concept

**When concept is missing:**
- Document: `missing / intentionally deferred / superseded / not in current scope`
- If "missing" → HIGH priority to restore
- If "deferred" → Document timeline
- If "superseded" → Document replacement
- If "out of scope" → Document reason
