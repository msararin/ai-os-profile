📊 WHAT I FOUND
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ROOT CAUSE: Stale Environment Variable Cache

BEFORE (Buggy):
  const allowedEmails = process.env.INTERNAL_ALLOWED_EMAILS?.split(',')...
  
  Problem: Module-level constant loaded once
  Impact: Cached across serverless invocations
  Result: Second login fails if initial load was empty/undefined

AFTER (Fixed):
  function getAllowedEmails() {
    return process.env.INTERNAL_ALLOWED_EMAILS?.split(',')...
  }
  
  Solution: Dynamic loading on each sign-in attempt
  Impact: Fresh environment variable read every time
  Result: Consistent authorization checks

TECHNICAL INSIGHT:
  Vercel Edge Functions use module caching for performance. While this
  is beneficial for most code, it can cause issues with environment-
  dependent initialization. Our fix ensures critical auth data is
  always fresh, preventing stale state bugs.
